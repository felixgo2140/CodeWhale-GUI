## v2.5.29 — 单聊/对比窗口防卡住

### 修复 / 优化
- **单聊窗口不再只靠侧栏缓存判断 turn 是否结束**:运行中会话现在每 4 秒直接查询当前 `/v1/threads/:id`,绕过 `/api/threads/all` 的 120 秒 stale cache;如果 SSE 漏掉 `turn.completed`,前端会自动收尾、解锁输入框并继续发送排队消息。
- **补齐单聊 turn 失败/中断事件监听**:单聊现在监听 `turn.failed` / `turn.interrupted`,不会因为只处理 `turn.completed` 而停在“运行中”。
- **统一 turn 状态判断**:兼容 `inprogress` / `in_progress` / `running` / `queued` 以及失败、中断、取消状态,侧栏状态点和输入框 running 状态不再各判各的。
- **多模型对比不再把长时间无 chunk 误判为卡死**:180 秒空闲只显示等待状态并直查当前 `/cmp/<provider>/v1/threads/:id`,不再直接把该栏收失败。
- **长任务进度更可见**:每个模型列新增运行条,显示阶段、工具步数、耗时和最近工具/审批/失败步骤;折叠的“过程”仍保留完整步骤。
- **补齐对比页取消控制**:总输入区新增“停止全部 / 清排队”,每列新增更明显的停止入口;停止时会尝试从当前 thread 找回最新 turn id 再发送 interrupt。
- **长连接更稳**:前端仍用 SSE,同时每 4 秒直查当前 thread 做兜底;后端代理长连接超时从 10 分钟放宽到 30 分钟,减少长模型/工具链任务被中间层切断。

### 备注
- 主要是网页和 GUI 代理改动,在线更新后刷新/重开对比窗口即可生效;二进制不变。

## v2.5.27 — Claude CLI 会话复用

### 修复 / 优化
- **Claude 订阅列不再每轮把整段历史拍扁重发**:Claude Code provider 现在用 CodeWhale thread UUID 作为 Claude CLI 会话 id,首轮走 `--session-id`,后续走 `--resume`,让 Claude 自己接住连续上下文,避免之前因为每次新开 `claude -p` + 全历史 stdin 导致慢、压缩、丢对话。
- **Claude 每次只发送当前用户输入**:不再把所有历史消息拼成 `user: ... / assistant: ...` 交给 CLI,减少 token/启动负担,也避免模型把旧上下文当成新的转述任务。
- **Claude 流式输出只渲染正文 delta**:改用 `stream-json` 解析,过滤 init/thinking/tool 等内部事件,最终无正文 delta 时再回退 result 文本。
- **增加 CLI 诊断逃生口**:`CODEWHALE_CLAUDE_BIN` 可指定本地 Claude CLI,`CODEWHALE_CLAUDE_DEBUG_LOG` 可记录调用模式/会话 id/长度,方便排查不同电脑的 Claude CLI 行为。

### 备注
- 本版包含补丁二进制更新;在线更新会下载新的 `codewhale-claude` / `codewhale-tui`。
- 这能明显减少重复历史带来的慢和上下文丢失,但 Claude CLI 仍有进程启动成本;若后面还要进一步提速,需要做常驻 Claude bridge。

- **切换到「非默认模型」对话不再连环卡**:`ensure_provider_server` 原来在等 app-server 就绪(最长 ~20s)的全程持锁,启动 A 会阻塞切到 B。现**就绪等待移出锁**,各 provider 后端并行启动互不阻塞;并发请求同一冷后端**只启一个进程**(启动锁去重)。
- **热切换近乎零延迟**:provider 后端「活着」状态缓存 15s,省去每次请求都探活的 HTTP 往返。
- **修「发送失败 HTTP 409 Thread already has an active turn」**:切回一条仍在跑的对话时,快照不会重放已发生的 turn → 界面误以为空闲、再发就 409。现切换时按 `turns` 末轮状态**同步"运行中"**(发送自动转排队);万一仍撞 409 也**优雅转排队**(上一轮完成后自动发),不再弹红字报错。
