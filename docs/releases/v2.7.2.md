# CodeWhale GUI v2.7.2

这是一次面向长任务、研究 Harness 和日常回复操作的完整稳定版。重点不是增加更多入口,而是让任务在上下文变长、附件仍在处理、Harness 子进程异常或 GUI 重载时,仍能可靠交付最终回复和产出文件。

## 重点更新

- **长线程主动整理上下文**:发送新消息前检查上下文压力,必要时先完成可恢复压缩再发送原消息;整理失败会明确提示,不会静默丢失输入。
- **Harness 产出可靠交付**:任务结束后会从 runtime、报告目录和 turn 产物中回收 Markdown/PDF,自动补到聊天窗口供预览、打开和下载。
- **异常任务不再假运行**:Harness 子进程退出或超过 30 分钟无进展且没有报告时,任务会进入明确失败态并保留手动恢复入口。
- **附件不阻塞输入**:上传后即可继续输入下一条,图片识别和文档提取在任务队列内完成;多条消息仍严格保持发送顺序。
- **Claude Desktop 自动接通**:除 PATH 和 `~/.claude/local/claude` 外,会发现 Claude Desktop 自带的 Claude Code CLI,并映射到稳定路径供 provider 调用。
- **回复复制与上下文引用**:每条模型回复可一键复制,也可用 `Command+Shift+C`;选中文字右键可复制选区、复制整条回复或附加到所属输入框。多模型对比不会把引用放错列。
- **界面与壳稳定性**:精简线程和消息操作按钮以减少误触;WKWebView 内容进程意外退出时自动恢复当前页面。
- **发布脚本更稳**:完整安装包构建支持相对输出路径并自动创建目标目录,本地验证和自动发布使用同一套流程。
- **Claude 运行时真正安装**:安装器会检查 CPU 架构,并把兼容的 `codewhale-claude` / `codewhale-tui` 部署到 `~/.codewhale-gui/bin`,不再只是把文件放在压缩包里。

## 下载与升级

### 已安装用户

在 CodeWhale 的「设置 → 更新」检查 v2.7.2。客户端会验证 Ed25519 签名和 SHA-256 后更新 GUI、Harness、原生 App及可选运行时。更新完成后退出并重新打开一次 CodeWhale。

### 新用户

下载 `codewhale-installer.tar.gz`,解压后双击 `install.command`。完整安装包包含 GUI、原生 macOS App、研究 Harness 和 Claude 订阅桥接二进制,不包含任何用户 API key、OAuth 凭据、会话数据或私有插件。

## 资产说明

- `codewhale-installer.tar.gz`:完整安装包。
- `gui-2.7.2.tar.gz`:GUI 与后端在线更新包。
- `harness-2.7.2.tar.gz`:研究 Harness 更新包。
- `CodeWhale.app.tar.gz`:双架构原生 macOS App。
- `codewhale-claude` / `codewhale-tui`:Claude 订阅桥接运行时;当前公开构建为 Apple Silicon `arm64`,Intel Mac 会安全跳过,不影响其它模型与 GUI。
- `manifest.json` / `manifest.json.sig`:签名更新清单。
- `SHA256SUMS`:公开资产校验值。

## 升级提示

- 正在运行的 provider 后端会在健康时被 GUI 接管,不会因普通重启被一刀切终止;显式重置、运行时更新或配置改变时才会重启对应后端。
- 自动上下文整理只在发送前检测到高风险时触发。界面不会在后台无限续跑,需要继续执行时仍由用户确认。
